Alpaca 行情代理 API

更新时间: 2026-05-18
代理地址: 35.88.155.223 | WS: 8767 | HTTP: 8768

这个代理让你用一个 token 就能获取美股实时行情和历史数据，不需要自己持有 Alpaca API key 或 ThetaData 账号。

新增: 期权历史数据现已接入 ThetaData 作为 Primary Provider，Alpaca 作为 Fallback。新增 /v1/options/open_interest 和 /v1/options/eod 端点。

快速开始

1. 拿到你的 token

在 http://35.88.155.223:3000/register.html 注册并选择套餐，等待管理员确认后即可自助生成 Token。不需要 Alpaca 账号。

2. 测试连通性

curl http://35.88.155.223:8768/health
# → OK

3. 连接实时行情 (WebSocket)

// 以股票为例，期权/crypto/news 换对应的 URL 即可
const ws = new WebSocket('ws://35.88.155.223:8767/stream');

ws.onopen = () => {
  ws.send(JSON.stringify({ action: 'auth', token: '你的token' }));
  // 收到 {"T":"success","msg":"authenticated"} 后
  ws.send(JSON.stringify({ action: 'subscribe', trades: ['AAPL'], quotes: ['AAPL'] }));
};

ws.onmessage = (msg) => {
  // 股票/期权/boats/overnight 的行情是 MsgPack 二进制，需要解码
  // crypto/news 是 JSON 文本
  console.log(msg.data);
};

4. 拉取历史 K 线 (HTTP)

curl -X POST http://35.88.155.223:8768/v1/history/bars \
  -H 'Content-Type: application/json' \
  -d '{"token":"你的token","symbol":"AAPL","start":"2026-05-13","end":"2026-05-15","timeframe":"1Min","limit":10}'

WebSocket 实时流

所有 WS 端点都是先连接 → 发送 auth → 收到确认 → 订阅 → 收数据。

⚠️ 帧编码（重要！）

流 编码格式
/stream (股票) MsgPack 二进制
/stream/options MsgPack 二进制
/stream/test MsgPack 二进制
/stream/boats MsgPack 二进制
/stream/overnight MsgPack 二进制
/stream/crypto JSON 文本
/stream/news JSON 文本

MsgPack 流必须用 msgpack 库解码，不能直接 JSON.parse。

Auth 消息

{"action": "auth", "token": "你的token"}

成功回复:

{"T": "success", "msg": "authenticated"}

订阅消息

股票 / 期权 / boats / overnight / test:

{"action": "subscribe", "trades": ["AAPL"], "quotes": ["AAPL"]}

Crypto:

{"action": "subscribe", "trades": ["BTC/USD"], "orderbooks": ["BTC/USD"]}

News:

{"action": "subscribe", "news": ["*"]}

限制

  • 不支持 * 通配符订阅（news 除外）
  • 股票代码不能带 .（BRK.B 不支持）
  • 不支持 WS bars / daily bars / LULD
  • Crypto 不支持 quotes

HTTP REST API

所有 HTTP 端点（/health 除外）都需要在 JSON body 里传 token:

{"token": "你的token", ...}

也可以传 HTTP header: Authorization: Bearer 你的token

参数 类型 必填 说明

期权历史 K 线 (双 Provider)

POST /v1/history/options/bars
参数 类型 必填 说明
token string ✅ 代理 token
symbols string ✅ 期权代码，逗号分隔，如 AAPL260522C00200000
start string ✅ 开始日期
end string ✅ 结束日期
timeframe string ❌ 默认 1Min，支持 1Min/5Min/15Min/30Min/1Hour/1Day
limit int ❌ 默认 10000
max_pages int ❌ 默认 100

💡 也可以用 symbol（单数），会自动转成 symbols。

数据路由逻辑: 该端点优先尝试 ThetaData (gRPC) 获取期权分钟/日终 OHLC，如果 ThetaData 无数据或异常，自动 fallback 到 Alpaca REST API。你不需要关心底层 Provider，传同样的参数即可。

期权持仓量历史 (ThetaData)

POST /v1/options/open_interest
参数 类型 必填 说明
token string ✅ 代理 token
symbol string ✅ 标的股票代码，如 AAPL
start string ✅ 开始日期 YYYY-MM-DD
end string ✅ 结束日期 YYYY-MM-DD
expiration string ❌ 到期日过滤，默认 * (全部)
strike string ❌ 行权价过滤，默认 * (全部)
right string ❌ C/P/both，默认 both
max_dte int ❌ 最大到期天数
strike_range int ❌ 行权价范围

返回每条记录的 open_interest 字段。

期权日终数据 (ThetaData)

POST /v1/options/eod

参数与 /v1/options/open_interest 完全相同。

返回字段包含: open, high, low, close, volume, open_interest, bid, ask, bid_size, ask_size 等日终快照。

鉴权错误

HTTP 状态 含义
200 成功
400 请求参数有误
401 token 无效（不在注册表中）
403 token 有效但没有该端点的权限
500 代理内部错误（上游 Alpaca 故障等）

常见问题

Q: Lean/QuantConnect 怎么配置？

ALPACA_PROXY_URL=ws://35.88.155.223:8767/stream
ALPACA_PROXY_TOKEN=你的token

Q: 历史数据走代理还是直连？
默认直连 Alpaca REST。如果直连失败且设置了 ALPACA_HISTORY_AUTO_FALLBACK=1，会自动切到代理。

Q: 期权历史 K 线为什么有时候快有时候慢？
/v1/history/options/bars 现在优先走 ThetaData (gRPC)，如果该期权在 ThetaData 中没有数据（如某些冷门合约），会自动 fallback 到 Alpaca REST。ThetaData 通常更快，fallback 会多一次网络往返。

Q: 我需要自己的 ThetaData 账号吗？
不需要。ThetaData 账号由代理服务器端持有，你只需要代理 token 即可调用所有端点。

Q: 期权代码格式？
标准 OCC 格式，如 AAPL260522C00200000 = AAPL 2026-05-22 Call $200。

Q: 为什么我收到的数据是乱码？
检查帧编码表 —— 股票和期权的 WS 流是 MsgPack，需要用 msgpack.unpackb() 解码。

Q: 支持哪些时间框架？
实时流无 bars；历史 K 线支持 1Min 5Min 15Min 1Hour 1Day。

附录: 部署信息

以下内容仅供维护参考。

  • 部署模式: Paper trading (IS_LIVE=false), Pro feed (IS_PRO=true)
  • 上游数据源:
  • 股票: Alpaca v2/sip
  • 期权实时流: Alpaca v1beta1/opra
  • 期权历史 K 线: ThetaData (primary, gRPC) → Alpaca v1beta1/options/bars (fallback)
  • 期权 OI/EOD: ThetaData option_history_open_interest / option_history_eod
  • Crypto: Alpaca v1beta3/crypto/us
  • News: Alpaca v1beta1/news
  • Boats: Alpaca v1beta1/boats
  • Overnight: Alpaca v1beta1/overnight
  • ThetaData SDK: v1.0.4 (Python 3.12, gRPC 直连云端，无需本地 Terminal)
  • Token 注册表: /app/users.json（file-backed，支持运行时增删）
  • 源代码: github.com/ikkisovi/Websocket-DataFeed-Proxy
